# Agent Architecture Design Prompt

Use the `agent-architecture` skill to design a reliable workflow for this agent system:

```text
Goal:
[What should the agent system accomplish?]

Inputs:
[What sources, files, APIs, tickets, or user messages does it read?]

Outputs:
[What should it produce?]

Tools:
[What tools, APIs, files, repos, or external systems can it touch?]

Risky actions:
[What actions are public, destructive, financial, production-facing, expensive, or hard to undo?]

Human approval:
[Where should a human approve before execution?]
```
